# ASCII Art Wallpaper for KDE Plasma

A native Plasma wallpaper plugin. The animation is 100% GPU: a fragment
shader evaluates a procedural field per screen cell, buckets it into a
brightness index, and samples the matching glyph from a small pre-baked
font-atlas texture. No video decode, no CPU string building, no extra
window/compositor tricks (works on X11 *and* Wayland since it's a real
Plasma wallpaper plugin, not an xwinwrap hack).

## Files

```
org.kde.asciiwallpaper/
├── metadata.json
└── contents/
    ├── ui/main.qml          # hosts the ShaderEffect, drives iTime
    ├── shaders/ascii.frag   # GLSL source (edit this to change the look)
    └── images/font.png      # generated glyph atlas (ramp: " .:-=+*#%@")
```

## 1. Compile the shader

Qt6's `ShaderEffect` needs a precompiled `.qsb` shader pack, not raw GLSL.
Run this **on your KDE machine** (needs `qt6-shadertools`, which ships the
`qsb` tool):

```bash
chmod +x build_shader.sh
./build_shader.sh
```

## 2. Install

```bash
chmod +x install.sh
./install.sh
```

Then: right-click Desktop → **Configure Desktop and Wallpaper** →
**Wallpaper Type** → **ASCII Art Wallpaper**.

Quick preview without applying it as your real wallpaper:
```bash
plasmawindowed org.kde.asciiwallpaper
```

## Tuning / performance knobs (all in `contents/ui/main.qml`)

- `cellPx` — ascii cell size in pixels. Bigger = fewer shader invocations
  per glyph lookup and chunkier look. 16px is a good default; go to 12 or
  10 for finer detail if you have GPU headroom.
- `targetFps` — capped at 30 by default. ASCII motion reads fine at 24-30fps
  and this halves GPU work vs uncapped 60fps for basically no visual loss.
  Plasma also stops the `Timer` (`root.visible` guard) when the wallpaper
  isn't actually being shown (e.g. screen off, virtual desktop switched away),
  so it costs nothing when hidden.
- `fg` / `bg` — colors.

## Changing the animation

Everything the wallpaper does is generated by the `field()` function in
`ascii.frag` — that's your "video" replacement. Right now it's a few
overlaid sine waves (a cheap plasma effect). Swap it for:
- flow/curl noise for a more organic look
- a second render pass sampling `iTime` + mouse position for interactivity
- an audio-reactive input if you pipe in an FFT texture from a small helper

The shader is intentionally simple (~10 sin/cos ops, one texture sample)
so it stays effectively free on any GPU from the last decade at 1080p/1440p
even at 60fps — the 30fps cap and 16px cells are headroom, not necessities.

## Notes / caveats

- Written for **Plasma 6 / Qt 6** (RHI shader pipeline via `qsb`). If you're
  still on Plasma 5, the shader needs to be written as a Qt5-style GLSL
  `ShaderEffect` (raw GLSL string in `fragmentShader`, no `qsb` compile step,
  and drop the `layout(std140, binding=0) uniform buf {...}` wrapper for
  plain `uniform` declarations instead) — ask if you need that version.
- If glyphs look blurry, make sure `smooth: false` stays set on the `Image`
  (nearest-neighbor sampling keeps the atlas crisp at the pixel level).
